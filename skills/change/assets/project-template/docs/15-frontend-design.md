本文档记录本项目的前端/界面设计方向。凡是涉及网页、小程序、浏览器插件、移动 App、桌面端、内部工具、后台系统、仪表盘或其他用户界面的项目，初始化阶段都必须先完成本文件初版。

如果项目完全没有用户界面，请在本文件中明确写明“本项目暂无前端界面”，并说明原因和后续是否可能补充。

## 8. 需要避免的问题

```text
不适合本项目的风格：
不应使用的布局：
不应使用的组件表达：
不应出现的交互：
可访问性风险：
移动端风险：
性能风险：
```

常见需要避免的问题：

1. 内部工具做成营销落地页。
2. 过度使用大面积渐变、装饰图形和低信息密度布局。
3. 卡片嵌套卡片。
4. 文本在按钮、表格、卡片、弹窗中溢出或遮挡。
5. 没有 loading、empty、error 状态。
6. 移动端只压缩桌面端布局，不重新组织信息。
