## 2. 升级原则

1. 升级只能补齐缺失文档和新增规则，不能覆盖已有项目内容。
2. 已有文档如需调整，必须保留原项目信息，并以增量方式追加。
3. 如果新模板与旧项目内容冲突，必须先说明冲突，再询问用户。
4. 升级完成后必须更新 `docs/development/10-current-status.md` 和本文档。
5. 升级不能替代正常需求澄清和技术决策。
