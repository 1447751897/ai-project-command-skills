# 安全策略

简体中文 | [English](SECURITY.md)

## 报告安全问题

如果你在 skill 模板或安装脚本中发现安全问题，请在 GitHub issue 中提供清晰的问题描述和复现步骤。

请不要在 issue 文本中包含真实密钥、访问 token、生产凭证、私钥或敏感项目数据。

## 范围

这个仓库包含本地 Codex skill 文件夹和安装脚本。Skills 本质上是指令和模板，不提供托管服务。

内置工作流中的重要安全边界：

- 不要硬编码密钥。
- 不要提交真实 token。
- 不要执行破坏性 Git 操作，除非用户明确要求。
- 即使在 `/goal --super` 模式下，涉及付费服务、生产数据删除、密钥、破坏性 Git 操作，以及明确安全/合规风险时，仍然必须停止并确认。
